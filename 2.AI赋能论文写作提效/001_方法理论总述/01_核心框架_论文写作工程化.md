# 核心框架：论文写作的工程化思维

## 一、核心命题

> 好论文不是等来的，是**设计**出来的

把论文写作看作一个工程项目——它有工序、有质量标准、有可以优化的流程。

---

## 二、论文写作 = 工程项目

| 工程要素 | 论文写作对应 |
|---------|------------|
| 交付物 | 可发表的论文 |
| 工序阶段 | 选题 → 调研 → 方法设计 → 实验 → 写作 → 投稿 → 修改 |
| 质量控制点 | 导师审阅 / Reviewer 意见 / 拒稿反馈 |
| 迭代优化 | Revision cycle、resubmit |
| 资源约束 | 时间、算力、文献获取能力 |

---

## 三、流程控制论：建立反馈环

工程系统的核心是**反馈控制**，论文写作同样适用：

```
[设定目标]
     ↓
  [执行工序]
     ↓
[AI 检测输出质量]  ← 传感器
     ↓
 [识别偏差]
     ↓
[人工判断 → 修正]  ← 控制决策
     ↓
  [再执行]
     ↑___________反馈环___________|
```

**三个角色定义：**
- **传感器**：AI 发现质量问题（逻辑漏洞、语言问题、文献覆盖不足）
- **执行器**：AI 工具完成具体工序
- **控制目标**：由研究者定义（发哪个期刊、证明什么论点）

---

## 四、工作分类：三层结构

```
        ┌──────────────────┐
        │    创造性工作     │  ← 人主导
        │  选题 / 假设提出  │     AI 作镜子
        ├──────────────────┤
        │    分析性工作     │  ← AI 辅助 + 人决策
        │  综述 / 方案对比  │
        ├──────────────────┤
        │    机械性工作     │  ← AI 全包
        │  格式 / 润色 / 引用│
        └──────────────────┘
```

### 机械性工作（规则明确、可重复）

AI 应全包的工作：
- 文献格式与引用规范整理
- 英文语法润色
- 期刊投稿格式套用
- 数据清洗、格式转换
- 摘要/关键词初稿生成
- Reviewer 回复信的格式化表达

### 分析性工作（有框架可循、需判断）

AI 辅助、人做决策：
- 文献综述的结构组织
- 实验方案对比
- 数据可视化方案选择
- 结果解读初稿
- Introduction 逻辑链梳理

### 创造性工作（需直觉与领域洞察）

人主导，AI 作"思维镜子"：
- 研究问题的选择
- 核心假设与 insight 提出
- 跨领域类比与迁移
- 对结果"意外性"的判断

---

## 五、关键认知转变

| 旧思维 | 新思维 |
|-------|-------|
| 我要写一篇好论文 | 我要设计一个产出好论文的流程 |
| 等灵感、等状态 | 分阶段执行，每阶段有明确输入输出 |
| AI 是作弊工具 | AI 是工序中的执行器，思想仍是我的 |
| 每篇论文重新开始 | 建立可复用的个人 SOP |

---

## 六、博士生的不可替代性

AI 越强，这三件事越值钱：

1. **会设计流程** — 知道用什么工具、如何组合
2. **会提出问题** — 选择真正值得研究的问题
3. **会判断质量** — 区分 AI 产出的好与坏

> 执行力可以被替代，**问题意识**不能。
