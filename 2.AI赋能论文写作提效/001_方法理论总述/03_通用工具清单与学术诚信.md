# 通用工具清单与学术诚信指南

## 一、核心工具清单（所有学科通用）

| 工具 | 核心用途 | 上手难度 | 获取 | 是否免费 |
|-----|---------|---------|------|---------|
| **Claude** | 写作润色、逻辑分析、代码、综合推理 | 低 | claude.ai | 免费可用 |
| **ChatGPT / GPT-4o** | 通用分析、草稿生成、格式处理 | 低 | chat.openai.com | 免费可用 |
| **NotebookLM** | 论文问答、跨文献分析、综述辅助 | 极低 | notebook.google.com | 完全免费 |
| **Elicit** | 文献检索与自动摘要 | 低 | elicit.org | 免费可用 |
| **Zotero** | 文献管理、引用格式自动化 | 低 | zotero.org | 完全免费 |
| **Semantic Scholar** | 学术文献搜索、引用网络分析 | 低 | semanticscholar.org | 完全免费 |
| **Connected Papers** | 文献引用关系可视化 | 极低 | connectedpapers.com | 免费可用 |
| **Grammarly** | 英文语法润色 | 极低 | grammarly.com | 免费可用 |
| **Cursor** | 数据分析脚本生成、代码调试 | 低-中 | cursor.sh | 免费可用 |
| **Perplexity** | 带来源引用的快速问答 | 低 | perplexity.ai | 免费可用 |

## 二、工具选用决策树

```
需要处理文字 / 写作？
  ├─ 英文润色 → Grammarly → Claude 精修
  ├─ 起草初稿 → Claude / GPT-4o
  └─ 逻辑检查 → Claude（扮演 Reviewer）

需要处理文献？
  ├─ 检索发现 → Elicit + Semantic Scholar
  ├─ 理解内容 → NotebookLM
  ├─ 管理引用 → Zotero
  └─ 关系分析 → Connected Papers

需要处理数据 / 代码？
  ├─ 写分析脚本 → Cursor
  ├─ 可视化 → Cursor + matplotlib / R
  └─ 数学推导验证 → Wolfram Alpha

需要快速查资料？
  └─ 带引用的问答 → Perplexity
```

## 三、学术诚信边界

### 各类使用场景判断表

| 使用场景 | 合规性 | 说明 |
|---------|--------|------|
| 用 AI 润色英文语言表达 | 通常允许 | 思想是你的，AI 修改了语言形式 |
| 用 AI 生成初稿后大量修改 | 通常允许 | 多数期刊允许，建议提前查期刊政策 |
| 用 AI 辅助文献综述归类 | 通常允许 | 属于信息处理，非观点生成 |
| 用 AI 分析数据、辅助统计 | 通常允许 | 方法部分说明使用工具即可 |
| 直接提交 AI 内容不注明 | 争议 / 部分禁止 | 主流期刊已要求披露 |
| 用 AI 编造数据或伪造引用 | 严格禁止 | 属于学术不端 |
| 用 AI 生成代码并使用其结果 | 通常允许 | 需验证代码正确性并理解原理 |

### 投稿前必做：查期刊 AI 政策

主流期刊立场（截至 2025 年）：
- **Nature / Science**：要求在方法部分披露 AI 使用情况，AI 不能列为作者
- **Elsevier 旗下期刊**：要求在致谢中说明 AI 辅助工具的使用
- **IEEE**：允许 AI 辅助，但作者对内容负全责
- **国内 CSSCI 期刊**：政策不统一，建议直接查期刊官网或联系编辑部

### 核心原则

> AI 是工具，思想是你的。
> 用 AI 生成草稿再修改，与用 Word 拼写检查没有本质区别。
> 但直接提交 AI 生成内容不加标注，是对编辑和读者的信息欺骗。

## 四、常见 Prompt 速查表

### 写作类

```
# 摘要生成
你是学术写作专家。以下是我论文的核心内容：[内容]
请生成符合[期刊类型]标准的英文摘要，150-200词，
结构：背景/方法/结果/结论

# 段落润色
请润色以下英文段落：学术风格、简洁、主动语态、消除重复
保留原意，不添加我没有说的内容：[段落]

# 逻辑检查
请扮演顶刊审稿人，审读以下内容，
找出逻辑跳跃、论据不足、表述模糊之处：[内容]
```

### 分析类

```
# 研究空白识别
你是[领域]资深研究者。研究方向：[方向]
请梳理已有研究方向、识别研究空白、推荐3个有价值的切入点

# 文献综述框架
基于研究主题：[主题]
生成4节文献综述框架，每节含：综述逻辑 / 文献类型 / 引出的空白

# Reviewer 预审
扮演[期刊]审稿人，针对以下框架提出3个主要质疑：[内容]
```

### 投稿类

```
# 期刊推荐
论文主题：[简述] / 方法：[简述] / 贡献：[简述]
推荐5个适合投稿的期刊，含：名称/影响因子/接收偏好/审稿周期

# Response Letter
以下是审稿人意见：[意见]
起草回复信框架：致谢开场 + 每条意见回应逻辑 + 修改说明
语气：自信专业，不卑微
```
