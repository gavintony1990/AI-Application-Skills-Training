# L02 从Chat到Agent：AI应用的下一个十亿美金机会

## 核心论点

> Chat 是 AI 的第一阶段：人问，AI 答。
> Agent 是 AI 的第二阶段：人给目标，AI 自己执行。
> 十亿美金的机会，在 Agent 里——但 99% 的创业者还在做 Chat 的事。

## Part 0 — 开场演示

现场展示：一个 AI Agent 完成以下完整任务流（5分钟）：
1. 接收指令："帮我研究竞争对手 X 的最新动态，整理成汇报"
2. 自动搜索网络信息
3. 提取关键内容
4. 结构化整理
5. 生成 Word 报告

> "这不是 ChatGPT。这是 Agent。区别是：你不需要每一步告诉它怎么做。"

## Part 1 — Chat vs Agent：本质区别

### 理解层次递进

```
Level 1：Chat（问答）
  输入：一个问题
  输出：一个答案
  特点：单轮交互，人全程参与

Level 2：高级 Chat（多轮对话）
  输入：一段需求
  输出：迭代完善的结果
  特点：多轮交互，人仍然是主导

Level 3：Copilot（辅助工具）
  输入：人的操作意图
  输出：建议 + 辅助执行
  特点：人做决策，AI 做执行

Level 4：Agent（自主执行）
  输入：目标（Goal）
  输出：完成的任务
  特点：AI 自主规划步骤、调用工具、处理异常
```

### Chat 的商业天花板

Chat 应用的问题：
- **替代品太多**：每家大模型都能做 Chat，差异化难
- **用户黏性低**：用户随时可以换一个 Chat 入口
- **变现逻辑弱**：用户为聊天付多少钱？
- **壁垒不存在**：没有数据飞轮，没有工作流锁定

### Agent 的商业飞轮

```
Agent 完成任务 → 产生工作流数据
      ↓
数据优化 Agent 表现
      ↓
Agent 更好 → 更深嵌入企业工作流
      ↓
切换成本极高 → 护城河形成
```

## Part 2 — Agent 的技术架构与能力边界

### Agent 的四个核心组件

```
┌─────────────────────────────────────────┐
│              AI Agent                   │
│                                         │
│  ┌─────────┐    ┌─────────────────┐    │
│  │  大脑    │    │   工具箱         │    │
│  │（LLM）  │←→  │ 搜索/代码/API   │    │
│  └─────────┘    │ 文件/邮件/浏览器 │    │
│       ↕         └─────────────────┘    │
│  ┌─────────┐    ┌─────────────────┐    │
│  │  记忆    │    │   规划能力       │    │
│  │（上下文）│    │ 任务分解/反思   │    │
│  └─────────┘    └─────────────────┘    │
└─────────────────────────────────────────┘
```

### 当前 Agent 的能力边界（2025年）

**能做好的**：
- 信息搜集与整理（研究类任务）
- 结构化数据处理
- 代码编写与调试
- 标准化流程执行（有明确规则的）
- 多步骤表单填写与文档生成

**还做不好的**：
- 需要物理世界交互的任务
- 高度非结构化的创意判断
- 需要强监管合规的最终决策
- 跨系统权限复杂的操作

### 创业者的关键判断

> 找那些"步骤明确、重复执行、人工执行成本高"的任务。
> 这就是 Agent 最先落地的场景。

## Part 3 — 五个垂直行业的 Agent 机会图谱

### 行业 1：法律与合规

**当前痛点**：合同审查、尽调、合规检查 → 律师时间最贵
**Agent 机会**：
- 合同审查 Agent：上传合同 → 自动标注风险条款 → 生成修改建议
- 尽调 Agent：给定目标公司 → 自动搜集公开信息 → 生成结构化报告
- 合规监控 Agent：实时监控法规变化 → 自动识别影响 → 推送合规建议

**已有玩家**：Harvey、Clio、Robin AI
**机会窗口**：中国本土法律 + PIPL/数据合规细分方向基本空白

---

### 行业 2：金融与投资

**当前痛点**：研究报告撰写、尽调、风控监控 → 分析师人力密集
**Agent 机会**：
- 行业研究 Agent：给定行业 → 自动搜集财报/新闻/政策 → 生成研究框架
- 财报分析 Agent：上传财报 → 自动提取关键指标 → 对比历史/同行
- 风控预警 Agent：实时监控持仓相关新闻 → 自动触发预警

**已有玩家**：BloombergGPT、Kensho、Arini
**机会窗口**：A股本土研究 + 中小私募机构的低成本研究工具

---

### 行业 3：医疗与生命科学

**当前痛点**：病历整理、文献检索、临床试验管理 → 医生和研究员时间宝贵
**Agent 机会**：
- 病历结构化 Agent：非结构化病历 → 标准化数据 → 诊断辅助
- 临床文献 Agent：给定疾病/药物 → 自动综述最新文献 → 推荐治疗方案参考
- 药物研发 Agent：靶点确定 → 自动检索类似结构/机制 → 生成研究思路

**已有玩家**：Nabla（欧洲）、Abridge、Insilico Medicine
**机会窗口**：中文医疗数据 + 中医/中西医结合的独特场景

---

### 行业 4：企业销售与市场

**当前痛点**：客户研究、个性化外联、CRM 数据录入 → 销售效率低下
**Agent 机会**：
- 销售情报 Agent：给定目标客户 → 自动搜集背景信息 → 生成个性化开场话术
- 邮件外联 Agent：批量生成个性化销售邮件 → 跟进序列自动化
- 竞争分析 Agent：实时监控竞品动态 → 自动生成对比销售材料

**已有玩家**：Clay、Apollo.io、Outreach
**机会窗口**：国内 ToB 销售流程的 Agent 化（企微 + CRM 深度整合）

---

### 行业 5：软件开发与 IT

**当前痛点**：代码 Review、测试用例编写、文档维护 → 开发效率瓶颈
**Agent 机会**：
- 代码 Review Agent：提交代码 → 自动 Review → 生成改进建议
- 测试 Agent：给定需求文档 → 自动生成测试用例 → 执行测试
- 运维 Agent：监控到异常 → 自动诊断原因 → 执行修复脚本

**已有玩家**：Cursor、Devin、GitHub Copilot Workspace
**机会窗口**：面向非 BigTech 的中小开发团队 + 国产替代

## Part 4 — 如何找到你的 Agent 机会

### 筛选框架：好的 Agent 场景具备什么？

```
✅ 任务步骤可以被明确描述（规则清晰）
✅ 重复执行频率高（每天/每周）
✅ 当前由人工执行且成本高
✅ 结果可以被量化验证（知道做对了没有）
✅ 有容错空间（错了可以被人 review 后修正）

❌ 需要情感判断（谈判、心理辅导）
❌ 法律责任无法转移
❌ 完全非结构化的创意工作
❌ 一次性任务（不值得开发 Agent）
```

### 验证流程：从想法到原型的最短路径

```
Step 1：找到一个具体任务（不是一个行业，是一个具体动作）
         例：不是"法律 Agent"，而是"NDA 合同的风险条款标注"

Step 2：手动完成这个任务 10 次，记录每次步骤
         → 发现规律，验证可以被标准化

Step 3：用现有工具（Claude/GPT + 工具调用）拼一个原型
         → 不要自己训模型，先用 API

Step 4：找 5 个真实用户，让他们用，观察哪里卡壳

Step 5：根据卡壳点迭代，不是根据你的想象迭代
```

### Agent 创业的三个常见误区

1. **误区一：Agent 越复杂越好**
   → 现实：最先成功的 Agent 都是单一任务、边界清晰的
   → 正确：从最小可用的单一任务 Agent 开始

2. **误区二：要做通用 Agent**
   → 现实：通用 Agent 是平台公司的战场（OpenAI/Anthropic）
   → 正确：做垂直 Agent，深入一个场景

3. **误区三：先建技术，再找场景**
   → 现实：技术已经够用，场景 PMF 才是核心挑战
   → 正确：先找到真实痛点，再用技术实现
